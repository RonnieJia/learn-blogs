    Swift面向协议编程

    vuePress-theme-reco 贾平辉    2015 - 2022

    Swift面向协议编程

    贾平辉 3/5/2021 iOS

    # 前言

    Apple在2015年的WWDC提出Swift是一种面向协议编程（Protocol Oriented Programming，简称POP）的语言，在Swift的源码中我们可以看到大量的协议。本文主要目的是带领大家简单认识面向协议编程。

    本文主要从以下这几点来简单介绍：

    1. Swift中的协议
    2. 为什么面向协议
    3. 协议扩展的一些注意点

    # 1. Swift中的协议

    Swift 标准库中有 50 多个复杂不一的协议，几乎所有的实际类型都是满足若干协议的。protocol 是 Swift 语言的基础，语言的其他部分正是在这个基础上组织和建立起来的。这和我们熟知的面向对象的构建方式很不一样。

    一个简单的Swift协议可以如下定义：

    protocol Eatable {
    var isHunger: Bool { get }
    func eatFoods()
}

    可以看到Swift协议就是一组属性或方法的定义，而如果某个具体类型想要遵守一个协议，那它需要实现这个协议所定义的所有这些内容。协议实际上做的事情不过是“关于实现的约定”。

    # 2. 为什么面向协议

    Swift和OC一样都是面向对象（OOP）的语言，同OC一样使用OOP编写代码能满足我们所有需求的开发。那为什么还需要引入POP呢？

    OOP是一种很好的抽象方式，但有些场景还存在一些缺陷。它无法为两个不同继承关系的类添加同一个方法。例如有个CustomeLabel和CustomButton分别继承与UILabel和UIButton，现在需要给这两个自定义的控件添加同一个方法func method()，OOP我们可能会使用以下几种方案：

    1. 同样的代码分别写到CustomeLabel和CustomButton中。这种方案不建议采用，应该尽量避免这种做法。
    2. 生成一个单独的类实现这个func method()方法，然后CustomeLabel和CustomButton引入这个类。这种方案引入了额外的依赖。
    3. 将func method()这个方法添加到共同的父类UIView中。这种方案会给UIView添加更多更多的功能代码导致分工不明确，并且UIView的一些其他子类并不需要这个方法。

    使用POP解决上述问题，首先我们需要定义一个协议：

    protocol Customable {
    func method()
}

    然后CustomLabel和CustomButton同时遵守这个协议，并实现func method()方法

    class CustomLabel: UILabel, Customable {
    func method() {
        print("This is customMethod")
    }
}

class CustomButton: UIButton, Customable {
    func method() {
        print("This is customMethod")
    }
}

    # 协议扩展

    这个方案func method()还是在CustomLabel和CustomButton中同时实现了，和OOP相比，并没有任何优势。但是Apple在Swift2.0引入为协议引入了一种新的特性，协议扩展，就是可以为协议提供一个默认实现。

    extension Customable {
    func method() {
        print("This is customMethod")
    }
}

    此时CustomLabel和CustomButton只需要遵守这个协议，就可以直接使用func method()这个方法了。

    此外，在Swift中Apple的建议是应该更多的使用值类型（结构体类型和枚举类型等），值类型是无法继承的，但是可以引入协议。

    # 协议继承

    协议可以从其他协议继承，然后在它继承的需求之上添加功能，因此可以提供更细粒度和更灵活的设计。

    # 协议组合

    类、结构体和枚举可以遵守多个协议，它们可以采用多个协议的默认实现。这在概念上类似于多继承。这种组合的方式不仅比将所有需要的功能压缩到一个基类中更灵活，而且也适用于值类型。

    总的来说，面向协议就是通过协议扩展、协议继承和协议组合的方式来设计编写需要的代码。

    # 3. 协议的一些注意点

    1. Swift协议中的属性：

      • 协议可以定义实例属性和类型属性(使用static)
      • 协议不指定属性是存储属性还是计算型属性，只指定属性名称和类型以及读写性
      • 协议指定属性的读取类型，使用的get和set，中间不能使用逗号
      • 协议总是使用var关键字来声明变量属性
      • 不能给协议属性设置默认值，因为默认值被看做是一种实现

    2. Swift协议中的方法：

      • 协议可以定义实例方法和类方法(使用static)
      • 协议定义函数时不能添加函数的实现，同时，传入的参数也不能使用默认参数
      • 如果协议定义的实例方法会改变实例本身，需要在定义的方法名前使用mutating；这使得结构和枚举能够遵循此协议并满足此方法要求。

    3. 协议扩展：

      • 添加现有的计算属性和方法，不能添加存储属性
      • 扩展不会增加协议的需求
      • 和泛型一样，协议也能使用where子句进行类型限制

    4. 协议扩展的分发规则

      •if 调用方法的变量被推断为协议类型
    •if 协议中声明了方法
        无论扩展中是否实现该方法，都调用运行时类型的实现(动态派发)
    •else
        直接调用默认实现 (静态派发)
•else
    调用该类型的实现
